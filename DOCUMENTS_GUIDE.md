# 📄 KAG - Загрузка документов в базу знаний

## 🚀 Как загрузить документы

### Способ 1: Через веб-интерфейс (рекомендуется)

1. Откройте **http://localhost:8000/documents**
2. **Перетащите файлы** в область загрузки или нажмите для выбора
3. Поддерживаемые форматы:
   - ✅ **PDF** (с извлечением текста по страницам)
   - ✅ **TXT, MD** (текстовые файлы)
   - ✅ **DOCX** (Word документы)
   - ✅ **CSV** (таблицы)
4. Документы автоматически:
   - Парсятся
   - Разбиваются на чанки
   - Векторизуются через Ollama embeddings
   - Сохраняются в Qdrant

### Способ 2: Через API

```bash
# Загрузка одного файла
curl -X POST http://localhost:8000/api/v1/upload/ \
  -F "file=@document.pdf"

# Пакетная загрузка
curl -X POST http://localhost:8000/api/v1/upload/batch \
  -F "file=@doc1.pdf" \
  -F "file=@doc2.docx" \
  -F "file=@data.csv"

# Проверка статуса
curl http://localhost:8000/api/v1/upload/{document_id}/status

# Список всех документов
curl http://localhost:8000/api/v1/upload/list

# Удаление документа
curl -X DELETE http://localhost:8000/api/v1/upload/{document_id}
```

## 📊 Что происходит при загрузке

```
1. Загрузка файла → /app/data/uploads/
2. Парсинг → извлечение текста и метаданных
3. Чанкинг → разбиение на фрагменты по 1000 символов
4. Векторизация → создание embeddings через Ollama
5. Сохранение → запись в Qdrant с метаданными
```

## 🔍 Как работает поиск (RAG)

После загрузки документов:
1. Пользователь задаёт вопрос в чате
2. Система ищет релевантные чанки в Qdrant
3. Найденные документы добавляются в контекст
4. LLM генерирует ответ на основе контекста
5. Ответ показывается с источниками

## 📁 Где хранятся файлы

- **Загруженные файлы**: `/app/data/uploads/`
- **Векторная база**: Qdrant (контейнер `kag-qdrant`)
- **Метаданные**: В памяти API (в будущем - БД)

## ⚙️ Настройки парсинга

Параметры чанкинга (можно изменить в коде):
- **chunk_size**: 1000 символов
- **chunk_overlap**: 200 символов

Embedding модель:
- **Модель**: qwen3-embedding:4b
- **Размерность**: 2560

## 🎯 Пример использования

1. Загрузите документы через http://localhost:8000/documents
2. Дождитесь статуса "✅ Готово"
3. Перейдите в чат http://localhost:8000
4. Задайте вопрос по содержимому документов
5. Получите ответ с источниками!
