# Сводка по всем сессиям проекта (kag-system-enhanced)

Этот документ объединяет контекст из всех недавних сессий, отражает текущее состояние проекта, реализованные фичи и принятые архитектурные решения.

## Текущее состояние
- **Ветка:** `feature/docling-integration` (используется как основная для новой логики обработки).
- **Сервер (production):** Внешний `37.204.20.233` (`qd.gostsecret.ru`), локально `192.168.50.18`.
- **Пользователь:** `yartsevn`.
- **Движок распознавания:** `Occular-ocr` (для русского текста) и `Docling` (для структуры/таблиц).
- **БД и поиск:** 
  - `PostgreSQL` (через SQLAlchemy, основная база: `config_store`, `DocumentRecord`).
  - `Qdrant` (векторный поиск, HNSW m=32, INT8).
  - `Neo4j` (Knowledge Graph).

## Реализованный функционал (последние сессии)
За несколько последних крупных сессий мы проделали огромный объём работы (>2000 сообщений в сумме).

### 1. Обработка документов и Чанки
- Реализована сквозная нумерация чанков (от `chunk_00001`).
- Исправлен поиск документов (`/details` теперь читает данные из in-memory `config_store` с приоритетом над Qdrant, что исправляет проблему, когда `get` по ID ничего не находил).
- Настроена синхронизация полей `document_type` и `recognized_title`.
- Добавлен `aiohttp` для стабильной работы анализатора и экстрактора (без него они падали молча).
- Чанки разбиваются, извлекается структура (Docling) + OCR (Occular). Векторизация идёт по батчам через Ollama (`nomic-embed-text` / `phi4-mini`).

### 2. Поиск и Knowledge Graph (KG)
- Реализован 3-режимный поиск: 
  1. По чанкам (Qdrant).
  2. По графу (Neo4j).
  3. По метаданным (PostgreSQL).
- Добавлен сырой Cypher query интерфейс для Neo4j.
- KG-роуты закрыты авторизацией, запросы параметризованы для безопасности.

### 3. Пользовательский интерфейс и Админка (UI)
- Строго соблюдается стиль **Linear Dark** (`#08090a`, акцент `#5e6ad2`, шрифт Inter, компактность A4 1:√2).
- Переработано навигационное меню: теперь 8 аккуратных пунктов, убраны дубли (например, единая кнопка «Логи»).
- Полностью переработана страница `/docker`: отображаются все контейнеры, диски (`du -sh`, `df -h`, `docker system df`), сеть и процессы.
- Добавлена живая вкладка «Обработка» в логах.
- Просмотрщик документов (Viewer) переведён на `PDF.js` (Canvas, клавиатура, зум), починены роуты `preview` и `thumbnails` (сделаны публичными для корректного отображения).

### 4. Документация и Деплой
- Техническая документация (Swagger) перенесена на `/api/docs`.
- Читаемая пользовательская документация доступна на `/docs`.
- Настроены авто-cleanup stale-записей при старте `document_service`.
- Все пароли генерятся криптографически и лежат в `config_store`, никаких plain-текстов в `.env` (кроме bootstrap).
- Загрузки сохраняются корректно в `/app/data/uploads` с нужными правами.

## Планы на будущие сессии
- Дальнейшая отладка графа знаний (Neo4j) и улучшение RAG.
- Оптимизация производительности `Occular-ocr` (на CPU он берёт 93.7%, Docling — тоже). Возможно, внедрение очередей для снятия нагрузки.
- Улучшение UI (поддержка fallback SVG при ошибках загрузки и дальнейшая полировка интерфейса).
