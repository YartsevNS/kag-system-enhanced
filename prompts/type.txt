содержат заголовок, реквизиты сторон, номер и дату — то есть всё,
по чему можно определить тип.

--- ПО ИМЕНИ ФАЙЛА ---
Имя файла — первый и быстрый источник информации:
  • «счёт», «invoice», «payment»           → invoice
  • «договор», «contract», «agreement»      → contract
  • «отчёт», «report», «analytics»         → report
  • «письмо», «letter»                      → letter
  • «паспорт», «passport», «удостовер»     → identity
  • «медицин», «больнич», «анализ»         → medical
  • «доверенность», «иск», «суд»           → legal
  • «приказ», «order»                       → order
  • «регламент», «политик», «инструкция»   → policy
  • «стандарт», «гост», «snip»             → standard
  • «финанс», «bank», «кредит»             → financial
  • «новость», «пресс», «news»             → news
  • форма, анкета, questionnaire            → form
  • свидетельство, сертификат, certificate → certificate
  • тех, technical, specification           → technical

--- ПО СОДЕРЖАНИЮ (первые строки документа) ---

  invoice — Счёт/Счёт-фактура
    Маркеры: «Счёт на оплату», «Счёт-фактура», «ИНН/КПП продавца»,
             таблица с номенклатурой, «Итого к оплате», «НДС»
    Структура: шапка с реквизитами → таблица позиций → итоговая сумма

  contract — Договор/Контракт
    Маркеры: «Договор №», «Стороны договора», «Предмет договора»,
             «Права и обязанности», «Ответственность сторон»
    Структура: преамбула → разделы → подписи сторон

  report — Отчёт/Аналитика
    Маркеры: «Отчёт», «Анализ», «Период», «Показатели»,
             «Выводы», «Рекомендации»
    Структура: заголовок → таблицы данных → заключение

  letter — Письмо/Обращение
    Маркеры: «Уважаемый», «Здравствуйте», «На №», «Сообщаем»
    Короткий документ, нет таблиц, есть обращения

  form — Форма/Анкета/Заявление
    Маркеры: «Заявление», «Анкета», «Заполните», строки с «___»
    Структура: поля для заполнения

  identity — Удостоверение личности
    Маркеры: «Паспорт», «Удостоверение», «Серия №», «Кем выдан»,
             «Дата рождения», «Место рождения»
    Структура: бланк строгой отчётности с серией и номером

  medical — Медицинский документ
    Маркеры: «Медицинская карта», «Диагноз», «Пациент», «Врач»,
             «Результаты анализов», «Назначения»

  legal — Юридический документ
    Маркеры: «Исковое заявление», «Решение суда», «Постановление»,
             ссылки на статьи законов, нормативные акты

  financial — Финансовый документ
    Маркеры: «Выписка», «Баланс», «Банковская», «Счёт №»,
             таблицы с суммами, банковские реквизиты

  certificate — Свидетельство/Сертификат
    Маркеры: «Свидетельство», «Сертификат», «Диплом», «Аттестат»
    Бланк с номером, датой выдачи, кем выдан

  order — Приказ/Распоряжение
    Маркеры: «Приказ №», «Распоряжение», «ПРИКАЗЫВАЮ», «Назначить»
    Структура: номер + дата → преамбула → постановляющая часть → подпись

  policy — Политика/Регламент
    Маркеры: «Политика», «Регламент», «Инструкция», «Порядок»,
             «Требования», «Должна соблюдаться»

  standard — Стандарт (ГОСТ/СНиП/ISO)
    Маркеры: «ГОСТ», «ISO», «СНиП», «СТО», «Стандарт организации»
             «Настоящий стандарт распространяется»

  technical — Технический документ
    Маркеры: «Техническое задание», «Спецификация», «Технические условия»,
             «Характеристики», «Параметры»

  news — Новость/Пресс-релиз
    Маркеры: дата + заголовок в первой строке, короткий текст,
             нет таблиц и реквизитов, публицистический стиль

 4. ФОРМАТ ВЫВОДА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Верни СТРОГО JSON (без markdown, без пояснений):
{{"entities":[{{"name":"ТИП","type":"document_type","confidence":0.95}}]}}

Где:
  • name — КЛЮЧ типа из списка (латиница, например "invoice", "contract")
  • confidence — уверенность от 0.0 до 1.0

Если тип определить не удалось → используй "other" с confidence=0.3
Если нужен НОВЫЙ тип, которого нет в списке → верни name на русском
  (например "Заявка") и type:"new_type"
