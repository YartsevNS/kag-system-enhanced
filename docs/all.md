### 💬 Вопрос
Дедупликация сущностей — зачем и как работает

### ✅ Решение
Дедупликация через MERGE вместо CREATE — это не косметика, а архитектурное решение:• Граф становится связным — через общие сущности можно перейти между документами• Экономия памяти — 1000 доков с «Ивановым» = 1 узел, не 1000• Быстрее поиск — MATCH (e:Entity {name: "Иванов"}) возвращает 1 запись• Аналитика — видно что «Иванов» в 150 документах (важный контрагент)Баг с chunk_id: раньше chunk_id = "chunk_00001" для всех доков → MERGE склеивал всё подряд. Исправлено на "{doc_id}_chunk_{i}".

### 💬 Вопрос
Баг: общие сущности — всего 3 при 130 документах

### ✅ Решение
Найден критический баг: chunk_id = "chunk_00001" был одинаковым для ВСЕХ документов. MATCH (c:Chunk {id: "chunk_00001"}) цеплял чанки всех документов → сущность «EUR/USD» была связана со 113 документами. Исправлено: chunk_id = "{document_id}_chunk_{i}". Старые сущности удалены, граф перестраивается.
