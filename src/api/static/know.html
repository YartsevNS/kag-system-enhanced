💬 Вопрос
№124. Celery prefork ломает SQLAlchemy connection pool
Первый save в БД проходит, второй (финальный со status=completed) падает молча: psycopg2.DatabaseError: error with status PGRES_TUPLES_OK. Ошибка пишется в DEBUG, не видна.
✅ Решение
Причина: Celery по умолчанию использует prefork pool — форкает процессы. Форк наследует SQLAlchemy connection pool родителя, соединения становятся битыми. Первый запрос переподключается (pool_pre_ping=True), второй использует другое битое соединение из пула. Fix: --pool=solo в CMD Dockerfile.worker — не форкает, один процесс, одно соединение. Коммит: af709ae. Очередь Redis очищена от старых задач (FLUSHDB).
