117|### 4.1. Загрузка документа

118|

119|```

120|POST /api/v1/upload/ (multipart/form-data)

121| │

122| ▼

123|┌──────────────────────────────────────┐

124|│ 1. upload_document() │ upload.py:20

125|│ • Валидация (SecurityValidator) │

126|│ • Сохранение файла на диск │ storage_service.py

127|│ → /app/user_data/uploads/ │

128|│ • Создание записи в config_store │ system_configs (PostgreSQL)

129|│ • Запуск process_document() в фоне│ BackgroundTasks

130|└──────────────────────────────────────┘

131| │ (фон)

132| ▼

133|┌──────────────────────────────────────┐

134|│ 2. process_document() │ document_service.py:205

135|│ Прогресс: 10% → 30% → 50% → 90% → 100%

136|│ │

137|│ [30%] Парсинг │

138|│ • document_parser.parse() │ parsers.py

139|│ • PDF → pdfplumber + Docling │

140|│ • DOCX → python-docx │

141|│ • OCR: Tesseract + Occular-ocr │ ocr_engine.py

142|│ • Возвращает segments[] │

143|│ │

144|│ [50%] Чанкинг │

145|│ • DocumentChunker.chunk() │ chunking.py

146|│ • Размер: 1000 символов │

147|│ • Перекрытие: 200 символов │

148|│ • chunk_id: chunk_00001, ... │

149|│ • chunk_seq: 1, 2, 3... │

150|│ │

151|│ [90%] Векторизация │

152|│ • EmbeddingsService │ embeddings_service.py

153|│ • nomic-embed-text (768d) │ → Ollama API

154|│ • Сохранение в Qdrant │ коллекция kag_documents

155|│ payload: {document_id, chunk_id, │

156|│ text, chunk_seq, ...} │

157|│ │

158|│ [100%] Завершение │

159|│ • Миниатюра (WebP, 500px) │

160|│ • Статус: completed │

161|│ │

162|│ [ФОН] Анализ первого чанка │

163|│ • DocumentAnalyzer │ document_analyzer.py

164|│ • LLM (phi4-mini) → title, type, │

165|│ summary, topics │

166|│ • Сохранение в config_store + │

167|│ обновление Qdrant payload │

168|└──────────────────────────────────────┘

169|```

405|### 8.2. GOST-шифрование (src/security/gost_crypto.py)

406|

407|Назначение: шифрование конфиденциальных данных (пароли, ключи) в соответствии с ГОСТ Р 34.12-2015 для совместимости с 152-ФЗ.

408|

409|Реализация:

410|- Шифрование: ГОСТ Р 34.12-2015 (Kuznyechik — 128-битный блочный шифр, Magma — 64-битный) через cryptography.hazmat.primitives.ciphers

411|- Хэширование: ГОСТ Р 34.11-2012 (Streebog — 256/512 бит) через hashlib

412|- Режим: CBC (Cipher Block Chaining)

413|- Ключ: хранится в /app/data/.encryption_key (256 бит), генерируется при первом запуске

414|- Использование: GOSTCrypto.encrypt(data), GOSTCrypto.decrypt(data)

415|

416|Класс GOSTCrypto:

417|```python

418|class GOSTCrypto:

419| def encrypt(self, plaintext: str) -> str # → base64

420| def decrypt(self, ciphertext: str) -> str # ← base64

421| def hash_streebog(self, data: bytes) -> str # hex-digest

422| def generate_key(self) -> bytes # 32 байта

423|```

424|

425|### 8.3. Валидация файлов (src/security/validator.py)

426|

427|SecurityValidator проверяет:

428|- Размер файла (макс. 50MB для PDF, 10MB для остальных)

429|- MIME-тип (whitelist: PDF, DOCX, TXT, MD, CSV)

## 11.1. Поток данных в Neo4j (Граф знаний)

### Общая схема

Данные попадают в Neo4j асинхронно после загрузки и векторизации документа.

Загрузка файла → Парсинг (Hybrid) → Чанкинг → Векторизация (Qdrant) → Построение графа (Neo4j)

### Ключевые файлы

- src/indexing/knowledge_graph.py: Сервис KnowledgeGraphService.

- src/indexing/entity_extractor.py: Класс EntityExtractor.

- src/api/services/document_service.py: Метод _build_knowledge_graph_async().

### Почему гибридный поиск может не работать?

- В системе нет документов (documents: 0).

- Граф знаний пуст (entities: 0, relations: 0).

- Гибридный поиск возвращает пустой результат, так как искать нечего.
