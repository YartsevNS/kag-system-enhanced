📖 KAG — Документация проекта

Knowledge Augmentation Generation — система управления документами с ИИ

🚀 Что такое KAG

KAG — это self-hosted система для загрузки, распознавания, классификации и поиска документов с использованием искусственного интеллекта. Она объединяет OCR, векторный поиск, граф знаний и языковые модели для работы с документами на русском языке.

📄 Загрузка

Перетащите PDF, DOCX, TXT, CSV — система распознает текст, создаст миниатюры и подготовит к поиску.

🔍 Поиск

Семантический поиск по содержимому документов. Задайте вопрос — найдёт ответ в чанках.

🕸️ Граф знаний

Автоматически извлекает сущности (люди, организации, даты, суммы) и строит связи между ними.

🤖 AI-анализ

Определяет тип документа, генерирует название и краткое содержание через языковую модель.

📋 Страницы системы

⬡ Дашборд

Главная страница. Сводка системы: количество документов, статус контейнеров, быстрые действия.

📄 Документы

Загрузка и управление. Drag-and-drop, карточки с A4-превью, фильтры по типу и статусу, прогресс-бар при загрузке. Клик по документу → просмотрщик.

🧩 Чанки

Просмотр чанков (фрагментов) документа. Пагинация, поиск, экспорт. Чанки пронумерованы по порядку.

🔎 Поиск

Семантический RAG-поиск. Векторный поиск по Qdrant + фильтры. Возвращает релевантные чанки с контекстом.

🕸️ Граф знаний

Neo4j. Поиск сущностей по типу, гибридный поиск (граф + вектор), статистика графа.

📋 Логи

Единый хаб: Docker-статус контейнеров, процесс-логи обработки документов (6 этапов), аудит действий, системная информация. Автообновление.

⚙️ Админ

Управление LLM-моделями (Ollama), загрузка/удаление, переключение активной модели, настройка внешних LLM для анализа.

🔧 Как это работает

Загрузка документа (6 этапов)

1. START      — получение файла, валидация, сохранение на диск
2. FIND_FILE  — поиск файла в /app/data/uploads/
3. PARSE      — извлечение текста (PDF→Docling+Occular, DOCX, TXT)
4. CHUNKING   — разбивка на чанки (1000 символов, перекрытие 200)
5. VECTORIZE  — эмбеддинги (nomic-embed-text, 768d) → Qdrant
6. COMPLETED  — миниатюра, метаданные, граф знаний

Фоновые процессы (после загрузки)

→ Анализатор: первый чанк → LLM → title, type, summary, topics
→ Граф знаний: чанки → LLM → сущности + связи → Neo4j
→ Сохранение в config_store (PostgreSQL) + Qdrant payload

🗄️ Инфраструктура

🐍 FastAPI

Основной сервер (kag-api). Uvicorn, порт 8000. 15 HTML-страниц, REST API.

🔍 Qdrant

Векторная БД. HNSW (m=32), INT8-квантование. Коллекция kag_documents, 768 измерений.

🕸️ Neo4j

Граф знаний. Узлы: Document, Chunk, Entity. Связи: HAS_CHUNK, MENTIONS, SIGNED_BY.

🗃️ PostgreSQL

Keycloak-DB. Таблицы: system_configs (config_store), documents, users.

⚡ Redis

Кэш + брокер Celery. maxmemory 2GB, политика allkeys-lru.

🦙 Ollama

Внешний сервер (192.168.50.41). Модели: phi4-mini (LLM), nomic-embed-text (эмбеддинги).

🔐 Безопасность

GOST-шифрование

ГОСТ Р 34.12-2015 Кузнечик Стрибог 152-ФЗ

Шифрование конфиденциальных данных. Библиотека cryptography==44.0.0. Ключ в /app/data/.encryption_key.

Аутентификация

JWT-токены (python-jose) + cookie kag_token. Keycloak OIDC (опционально). Casbin RBAC. Аудит-лог.

📦 OCR и парсинг

Движки распознавания

Occular-ocr — быстрый русский OCR (93.7% точность, CPU)
Tesseract — стандартный OCR (русский язык)
Docling — разметка и таблицы (CPU only)

Правило: Docling = структура, Occular-ocr = русский текст.
