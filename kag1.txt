# 📘 ПРОМПТ: АРХИТЕКТУРА И РЕАЛИЗАЦИЯ ПРОЕКТА `KAG` (KNOWLEDGE AUGMENTATION GENERATION)

## 🔹 1. ЦЕЛЬ И ОБЛАСТЬ ПРИМЕНЕНИЯ
Создать производственную систему `kag`, полностью заменяющую классический RAG-пайплайн. Система должна обеспечивать многоагентную обработку знаний, планирование задач, централизованный API-шлюз, веб-интерфейсы для администратора и конечного пользователя, а также модуль оценки качества генерации. Архитектура проектируется под горизонтальное масштабирование, обработку десятков ГБ мультимодальных документов и строгое соответствие требованиям 152-ФЗ.

## 🔹 2. ТЕХНОЛОГИЧЕСКИЙ СТЕК
| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Backend** | Python 3.11+, FastAPI, Pydantic V2 | Высокая производительность, асинхронность, нативная поддержка MCP и OpenTelemetry |
| **Векторная БД** | Qdrant (Cluster Mode) | Эффективное масштабирование, гибридный поиск, поддержка payload-фильтров для PDF/аудио |
| **Очереди/Планировщик** | Celery + Redis (или RabbitMQ), APScheduler | Асинхронная пакетная индексация, повторные попытки, cron-подобное планирование |
| **Кэширование** | Redis (с поддержкой GOST-хэшей для ключей) | Снижение задержки, сессии, rate-limiting |
| **Веб/Прокси** | Nginx + Auth-Request модуль | Статика, обратный прокси, терминация TLS, интеграция с Keycloak |
| **IdP / Auth** | Keycloak (OIDC/SAML/LDAP) + Casbin (RBAC) | Единая точка входа для AD (Windows) и FreeIPA (Linux), ролевая модель |
| **LLM / Инференс** | vLLM /  | Оптимизированный вывод, поддержка квантизации, пакетная обработка |
| **Мониторинг** | Prometheus + Grafana + Loki + OpenTelemetry Collector | Наглядные дашборды, трассировка запросов, экспорт в SIEM через Syslog/Fluent Bit |
| **Сборка/CI-CD** | GitLab CI + Docker Hub + Trivy + Syft | Автоматизация, сканирование уязвимостей, генерация SBOM |
| **MCP-сервер** | FastAPI-MCP (JSON-RPC over HTTP/SSE) | Стандартизированный интерфейс для подключения внешних клиентов и тестирования |

## 🔹 3. АРХИТЕКТУРА И ИНФРАСТРУКТУРА (DOCKER COMPOSE)
- Создать `docker-compose.yml` с профилями: `dev`, `prod`, `monitoring`.
- Каждый сервис: healthcheck, restart policy `unless-stopped`, лимиты ресурсов (`cpus`, `memory`), монтирование томов для персистентности.
- GPU-поддержка: `deploy.resources.reservations.devices: - driver: nvidia, count: all, capabilities: [gpu]`.
- Сетевая модель: изолированная bridge-сеть `kag_internal`, Nginx как точка входа с `proxy_pass` и `auth_request` к Keycloak.
- Масштабирование: поддержка `docker compose up --scale worker=3`, подготовка к миграции на Docker Swarm/K8s (stateless-компоненты вынесены в отдельные сервисы).

## 🔹 4. БЕЗОПАСНОСТЬ И СООТВЕТСТВИЕ 152-ФЗ
- **Шифрование в покое:** Настройка `dm-crypt`/LUKS2 с алгоритмом ГОСТ Р 34.12-2015 (через `gost-engine`) для томов данных. При необходимости — прикладное шифрование полей БД через `gostcrypto`.
- **Шифрование в транзите:** TLS 1.3 с отключением устаревших cipher suites. HSTS, OCSP Stapling.
- **Аутентификация:** LDAP-бинд к AD/FreeIPA, синхронизация групп, маппинг в Casbin-политики (`role, user, resource, action`).
- **Аудит:** Логирование всех действий аутентификации, изменений политик, запросов к LLM. Формат JSON, передача в Loki → SIEM.
- **Скан уязвимостей:** Интеграция `Trivy` в CI, блокировка сборки при `CRITICAL`/`HIGH`.
- **Реестр пакетов:** Автоматическая генерация SBOM через `syft dir:/app`, парсинг в JSON, вывод в админ-панели `/admin/dependencies`.

## 🔹 5. ОБРАБОТКА ДАННЫХ И МАСШТАБИРОВАНИЕ
- **Ингестия:** Конвейер распаковки PDF (с OCR для таблиц/изображений через `unstructured` или `Marker`), транскрипции аудио (`Whisper`), чанкинг с сохранением метаданных.
- **Пакетная индексация:** Celery-воркеры с приоритетными очередями, ретраи с экспоненциальной задержкой, поддержка частичного обновления.
- **Векторный поиск:** Гибридный режим (dense + sparse), фильтрация по типу документа, автору, дате, тегам.
- **Кэширование ответов:** Redis с TTL, инвалидация при обновлении документов, хэширование промптов по ГОСТ-совместимому алгоритму для ключей.

## 🔹 6. МОДУЛЬ ОЦЕНКИ КАЧЕСТВА
- Автоматические метрики: faithfulness, relevance, hallucination rate (через LLM-судью или классификатор).
- Ручная оценка: интерфейс администратора с аннотированием, экспорт в CSV/JSON.
- A/B-тестирование версий промптов/моделей, логирование результатов в Prometheus.
