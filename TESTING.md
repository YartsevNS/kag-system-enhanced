### SSH подключения

**Все пароли шифруются перед сохранением!**

- ✅ GOST шифрование для защиты конфиденциальных данных
- ✅ Файл настроек: `/app/data/ssh_connections.json` (зашифрован)
- ✅ Ключ шифрования: `/app/data/.encryption_key` (права 600)
- ✅ Пароли НЕ передаются обратно в API

### Настройка SSH подключения

1. Откройте http://localhost:8000/admin
2. Найдите секцию "🔐 Настройки SSH подключения к Ollama"
3. Заполните поля:
   - **IP адрес**: 192.168.50.41
   - **SSH порт**: 22
   - **SSH пользователь**: nick
   - **SSH пароль**: ваш пароль
   - **Порт Ollama API**: 11434
4. Нажмите "💾 Сохранить настройки"
5. Нажмите "🔍 Тест подключения" для проверки

## 📱 Что тестировать

### ✅ 1. Чат с AI

1. Откройте http://localhost:8000
2. Введите сообщение в поле ввода
3. Нажмите Enter или кнопку ➤
4. Дождитесь ответа (первый запрос может занять 1-2 минуты пока модель загружается)
5. Проверьте что:
   - Ответ получен
   - Модель отображается правильно (phi4-mini:latest)
   - Индикатор статуса зелёный

### ✅ 2. Управление моделями (Админ-панель)

1. На странице чата нажмите **"⚙️ Настройки"**
2. Откроется страница управления моделями http://localhost:8000/admin
3. Проверьте:
   - ✅ Статус Ollama сервера (должен быть зелёный)
   - ✅ Активная LLM модель
   - ✅ Активная Embedding модель
   - ✅ Список загруженных моделей Ollama

### ✅ 3. Смена модели

1. На странице админки выберите другую модель из выпадающего списка
2. Нажмите "Применить"
3. Вернитесь к чату (кнопка "← Назад к чату")
4. Проверьте что модель в шапке чата обновилась
5. Отправьте сообщение и проверьте ответ

### ✅ 4. Загрузка новой модели

1. На странице админки введите название модели:
   - `llama3:8b` - хорошая общая модель
   - `qwen2:7b` - хорошая модель для русского языка
   - `mistral:latest` - быстрая модель
2. Нажмите "Загрузить модель"
3. Дождитесь завершения загрузки (может занять несколько минут)
4. После загрузки выберите её из списка и примените

### ✅ 5. История чатов

1. Отправьте несколько сообщений в чате
2. Нажмите "➕ Новый чат" для создания новой сессии
3. Проверьте что история сохраняется в боковой панели
4. Кликните на сессию чтобы загрузить историю

## 🔧 API Endpoints для тестирования

### Health Check
```bash
curl http://localhost:8000/api/v1/health
```

### Chat API
```bash
curl -X POST http://localhost:8000/api/v1/chat/ \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"Привет! Как дела?"}]}'
```

### Статус моделей
```bash
curl http://localhost:8000/api/v1/admin/models/status | python3 -m json.tool
```

### Список LLM моделей
```bash
curl http://localhost:8000/api/v1/admin/models/llm | python3 -m json.tool
```

### Список Embedding моделей
```bash
curl http://localhost:8000/api/v1/admin/models/embeddings | python3 -m json.tool
```

### Все модели Ollama
```bash
curl http://localhost:8000/api/v1/admin/models/ollama-models | python3 -m json.tool
```

### SSH подключения

#### Получить настройки (без паролей)
```bash
curl http://localhost:8000/api/v1/admin/models/ssh-config | python3 -m json.tool
```

#### Сохранить настройки (пароль шифруется)
```bash
curl -X POST http://localhost:8000/api/v1/admin/models/ssh-config \
  -H "Content-Type: application/json" \
  -d '{
    "host": "192.168.50.41",
    "port": 22,
    "username": "nick",
    "password": "ваш_пароль",
    "ollama_port": 11434
  }' | python3 -m json.tool
```

#### Тест SSH подключения
```bash
curl -X POST http://localhost:8000/api/v1/admin/models/ssh-test | python3 -m json.tool
```

#### Перезапуск Ollama (с сохранёнными настройками)
```bash
curl -X POST http://localhost:8000/api/v1/admin/models/restart-ollama | python3 -m json.tool
```

### Переключение модели
```bash
curl -X POST http://localhost:8000/api/v1/admin/models/switch-llm \
  -H "Content-Type: application/json" \
  -d '{"backend_type":"ollama","model_name":"llama3:8b"}'
```

### Загрузка модели
```bash
curl -X POST http://localhost:8000/api/v1/admin/models/pull \
  -H "Content-Type: application/json" \
  -d '{"model_name":"llama3:8b"}'
```

## 🐛 Возможные проблемы

### "Таймаут запроса к Ollama"
**Причина:** Модель не загружена в память Ollama
**Решение:** Подождите 2-3 минуты, модель загрузится автоматически

### "Модель не найдена"
**Причина:** Модель не установлена на сервере Ollama
**Решение:** Загрузите модель через админ-панель

### "Ошибка подключения"
**Причина:** Сервер Ollama недоступен
**Решение:** Проверьте что 192.168.50.41:11434 доступен
